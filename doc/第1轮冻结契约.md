# Go2-X5 第 1 轮冻结契约

## 1. 目的

本轮只冻结接口与边界，不做大规模运行时重写。

冻结范围：

- typed protocol 边界
- deploy manifest 边界
- supervisor 模式机边界
- coordinator / unitree adapter / arm adapter 模块边界


## 2. 冻结结论

### 2.1 控制范式

- PPO 只输出 `12` 维腿部动作
- arm 由 external pipeline 控制
- gripper 只保留接口占位
- 不允许回到 whole-body PPO action 语义

### 2.2 数据面

- 主数据面：typed IPC
- ROS2：mirror / 运维面
- ROS2 不是控制真值源

### 2.3 配置面

- `deploy/go2_x5_real.yaml` 是默认真值源
- 旧 YAML / launch / CLI 只允许作为 override
- 启动必须打印 manifest path / hash / protocol version / policy id


## 3. typed protocol 冻结清单

### 3.1 冻结的消息类型

- `BodyStateFrame`
- `ArmStateFrame`
- `OperatorCommandFrame`
- `DogPolicyObservationFrame`
- `DogPolicyCommandFrame`
- `ArmCommandFrame`
- `BodyCommandFrame`
- `HybridDiagnosticFrame`
- `ModeEventFrame`

### 3.2 冻结的公共头字段

所有 typed frame 必须包含：

- `magic`
- `version`
- `msg_type`
- `seq`
- `source_monotonic_ns`
- `publish_monotonic_ns`
- `source_id`
- `mode`
- `validity_flags`
- `fault_flags`
- `payload_bytes`

### 3.3 冻结的关键语义

- 所有 command frame 必须包含 `command_expire_ns`
- `DogPolicyCommandFrame.action_dim` 固定为 `12`
- `BodyCommandFrame.joint_count` 固定为 `12`
- `ArmCommandFrame.joint_count` 固定为 `6`
- `ArmStateFrame` 必须包含：
  - `q`
  - `dq`
  - `tau`
  - `q_target`
  - `tracking_error`
  - `backend_age_us`
  - `transport_age_us`
  - `target_seq_applied`

### 3.4 第 1 轮不激活的扩展

- `arm_tracking_error_obs`
- `arm_command_delta_obs`

原因：

- 当前部署轮次先冻结为 `policy.observation_dim = 260`
- 若启用上述扩展，必须同步升级模型输入维度与 `policy_id`

### 4.3 P0 运行时键补充冻结

以下键从本轮开始视为 deploy manifest 的强约束，不再允许长期停留在 launch 硬编码层：

- `arm_adapter.arm_cmd_topic`
- `arm_adapter.arm_state_topic`
- `arm_adapter.arm_joint_command_topic`
- `ops.bridge_rmw_implementation`
- `ops.go2_rmw_implementation`

修正原因：

- 上一版冻结清单遗漏了这组 P0 运行时键，导致主路径仍存在 manifest 之外的默认值来源
- 这不属于功能扩展，而是把已经存在的运行时语义正式纳入 deploy contract

### 4.3 启动校验

启动时必须拒绝进入 `READY` 的条件：

- `policy.action_dim != 12`
- `robot.leg_joint_count != 12`
- `robot.arm_joint_count != 6`
- `arm_adapter.background_send_recv != true`
- `arm_adapter.servo_rate_hz < coordinator.rate_hz`
- `body_adapter.command_rate_hz != coordinator.rate_hz`
- `policy.policy_rate_hz > coordinator.rate_hz`


## 5. supervisor 模式机冻结清单

### 5.1 冻结的状态名

- `BOOT`
- `PROBE`
- `PASSIVE`
- `READY`
- `RL_DOG_ONLY_ACTIVE`
- `MANUAL_ARM`
- `DEGRADED_ARM`
- `DEGRADED_BODY`
- `SOFT_STOP`
- `FAULT_LATCHED`

### 5.2 冻结的关键转移

- `BOOT -> PROBE`
- `PROBE -> PASSIVE`
- `PROBE -> FAULT_LATCHED`
- `PASSIVE -> READY`
- `READY -> RL_DOG_ONLY_ACTIVE`
- `READY -> MANUAL_ARM`
- `RL_DOG_ONLY_ACTIVE -> DEGRADED_ARM`
- `RL_DOG_ONLY_ACTIVE -> DEGRADED_BODY`
- `RL_DOG_ONLY_ACTIVE -> SOFT_STOP`
- `RL_DOG_ONLY_ACTIVE -> FAULT_LATCHED`
- `DEGRADED_ARM -> READY`
- `DEGRADED_BODY -> READY`
- `DEGRADED_ARM -> SOFT_STOP`
- `DEGRADED_BODY -> SOFT_STOP`
- `FAULT_LATCHED -> PASSIVE`

### 5.3 冻结的阈值

- body stale: `50ms`
- arm stale: `50ms`
- policy stale: `100ms`
- arm tracking error 持续超限窗口: `200ms`
- degraded timeout: `1~2s`


## 6. 模块边界冻结清单

### 6.1 `hybrid_motion_coordinator`

负责：

- 消费 `DogPolicyCommandFrame`
- 消费 `ArmCommandFrame` 上层意图
- 消费 body / arm / operator state
- 统一限幅
- 统一过期判断
- 统一 tracking error / drift anchor 判断
- 输出最终 `BodyCommandFrame` 与 `ArmCommandFrame`

不负责：

- 直接读写 DDS
- 直接调 ARX SDK
- 自己定义模式机

### 6.2 `unitree_adapter`

负责：

- 读 `rt/lowstate`
- 写 `rt/lowcmd`
- 产出 `BodyStateFrame`
- 消费 `BodyCommandFrame`

不负责：

- PPO 推理
- arm 逻辑
- 模式机决策

### 6.3 `arm_adapter`

负责：

- 对接 `arx5-sdk`
- 保持 arm `500Hz` 内部 servo
- 消费 `ArmCommandFrame`
- 产出 `ArmStateFrame`

不负责：

- 底盘稳定控制
- PPO 推理
- 模式机决策


## 7. 第 1 轮禁止事项

- 不新增协议类型之外的私有控制消息
- 不新增 deploy manifest 之外的临时真值键
- 不修改 canonical 模式名
- 不在旧 whole-body action 兼容路径上继续扩展语义
- 不做大规模运行时重写


## 8. 下一轮进入条件

只有当以下 3 点同时成立，才进入并行实现轮次：

- manifest contract 已冻结
- typed protocol contract 已冻结
- mode machine contract 已冻结
